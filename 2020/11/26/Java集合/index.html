Java集合

集合（Collection）概述

Java集合定义了两种基本的数据结构：

  1. Collection：表示一组对象的集合；
  2. Map：使用kay-value存储，表示对象间的一系列映射或关联关系。

数组和集合的区别

  1. 容纳元素的区别
  • 数组：可以存储基本数据类型和引用数据类型
  • 集合：只能存储引用数据类型（如果存储的是Int类型，会自动封装为Integer）
  1. 长度的区别
  • 数组：长度是固定的，需要提前声明
  • 集合：长度可变

List，Set，Map三者的区别

  1. List：有序的对象集合。列表中每个元素有特定的位置，元素可以是重复的。
  2. Set：无序的对象集合。元素不可重复。
  3. Map：无序的对象集合。使用 key-value 形式存储对象，其中 key 无序且不可重复，value 无序可重复。

List集合总结

ArrayList:

  • 底层使用数组实现
  • 初始化时，默认容量为10，每次扩容的时增加原有容量的一半
  • 进行增删操作时，需要数组的拷贝复制（navite方法由C/C++实现）

LinkedList：

  • 底层实现使用双向链表（方便向前遍历）

Vector：

  • 底层使用数组实现，现在被ArrayList替代，原因如下：
    • 所有方法都是同步的，性能有所损失
    • 初始容量为10，超过时以100%的比率增加，会比ArrayList消耗更多内存

总的来说，查询多用 ArrayList ，增删多用 LinkedList。

ArrayList的增删慢是大多数情况，在下列极端情况时，增删速度要优于LinkedList：

  1. 如果一直使用 add() 增加元素（每次都增加到末尾）
  2. 一直删除 末尾元素 时（不需要移动位置）
  3. 删除 中间位置的元素 时
