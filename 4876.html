异常

发表于2021-12-20|更新于2022-04-11|Java

概述

  • Throwable
    所有异常都是继承自 Throwable

    • Error
      系统内部错误或资源耗尽，不应该抛出此类异常。
    • Exception
      应用程序级别的异常，这类异常必须捕捉
      • RuntimeException
        继承自 Exception，程序错误导致的异常，通常有以下几类：
        • 错误的类型转换
        • 数组越界
        • null指针
      • 其他异常
        不是派生于 RuntimeException 的异常，通常有：
        • 试图在文件尾部后面读取数据
        • 打开一个不存在的文件
        • 根据给定的字符串查找 class，但是并不存在

捕获异常

基本方法

在 try 部分中的代码抛出 catch 里声明的异常后，将跳过剩余代码块，执行 catch 部分的代码。

try - catch - finally 语句

三种情况会执行 finally 语句：

  1. 代码没有异常，则会执行标注的：1、2、5、6 四处代码；

  2. 代码有异常:

    • （标注的 一 部分报错）且被 catch 捕获，则会执行：1、3、4、5、6 五处代码；
    • （标注的 一、二 部分报错），异常将会抛回给这个方法的调用者，执行：1、3、5 三处代码；

  3. 代码抛出异常（标注的 一 部分报错），但没有被 catch 捕获，则会执行：1、5 两处代码。

try - catch - resource 语句

尽量使用该语句、或者 finally 中关闭资源，不要再 try 语句块中关闭。

throw 和 throws

  1. throws 关键字用于声明异常，它的作用和 try-catch 相似；而 throw 关键字用于显式的抛出异常。

  2. throws 关键字后面跟的是异常的名字；而 throw 关键字后面跟的是异常的对象。

  3. throws 关键字出现在方法签名上，而 throw 关键字出现在方法体里。

  4. throws 关键字在声明异常的时候可以跟多个，用逗号隔开；而 throw 关键字每次只能抛出一个异常。

实践

  1. 尽量不要捕获 RuntimeException，应该使用预先检查的方式规避错误；

  2. 尽量使用 try-with-resource 来关闭资源；

  3. 不要捕获 Throwable

    可能会捕获到程序无法处理的错误，例如：jvm 无法申请到足够的内存空间

  4. 不要在 finally 中 return

    try 块中的 return 语句执行成功后，并不会马上返回，而是继续执行 finally 块中的语句，如果 finally 块中也存在 return 语句，那么 try 块中的 return 就将被覆盖。 ———— 《阿里 Java 开发手册》
