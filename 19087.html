条件语句

v-for 具有比 v-if 更高的优先级，应避免 v-if 和 v-for 用在一起 3。

v-if

条件判断使用 v-if 指令：

3.1.1 v-else

可以使用 v-else 表示 else 语句，必须要跟在 v-if 后面，否则不会识别。

3.1.2 v-else-if

在 2.1.0 版本中新增，用作 v-if 的 v-else-if 语句，可以连续使用。

3.1.3 <template> 元素中使用 v-if

因为 v-if 指令必须用在一个元素上，如果需要切换多个元素，就需要使用 <template> 元素。

将 <template> 元素视为一个不可见的包裹元素，并使用 v-if 指令，最终渲染结果不包含该元素。

v-show

用法和 v-if 类似，但是带有该指令的元素会保留在 DOM 中。只是切换元素的 CSS property display。

v-show 指令不支持 template 和 v-if。

v-if 和 v-show

  • v-if

    • 是惰性的。如果初始条件为 false，则什么都不做；直到条件第一次变为 true 时，开始渲染。

    • “真正”的条件渲染。它会确保切换过程中，条件块内的事件监听器和子组件适当地销毁和重建。

  • v-show

    • 不论初始条件是什么，始终会渲染元素，只根据条件切换 CSS 属性。

一般来说，v-if 切换开销高，v-show 初始渲染开销高。开发时可以根据情况选用。
