一、Redis基础

1. 什么是Redis?

  • 1.1 NoSQL介绍

    • 概念：非关系型数据库，设计初衷是解决大数据时代的高并发、多结构化数据存储问题。
    • 特点：灵活的数据模型、高可扩展性、分布式架构、高性能。
    • 与SQL的区别：数据模型、查询语言、扩展性、一致性、使用场景。

  • 1.2 Redis介绍

    • 官网：https://redis.io
    • 背景：由Salvatore Sanfilippo开发，解决LLOOGG项目实时统计需求。
    • 数据类型：键值对，支持String、Hash、List、Set、Sorted Set等。
    • 发展阶段：初创（2009）、成长（2010-2013）、成熟（2014-2017）、扩展（2018至今）。

  • 1.3 Redis优势

    • 多语言客户端、原子性操作、高性能、多样化数据结构、持久化机制、高可用与分布式支持、简单易用、生态支持。

  • 1.4 Redis应用场景

    • 缓存、排行榜、计算器/限速器、好友关系、消息队列、Session共享、分布式锁、地理空间数据处理、实时分析。

  • 1.5 Redis与Memcached的区别

    • 数据类型、持久化、内存管理、复制与高可用、分布式支持、更新与社区支持、底层模型、使用场景。

2. Redis的工作原理

  • 2.1 基于内存的数据存储
    • 存储在RAM中，读写速度极快（读11万/s，写8.1万/s）。
  • 2.2 数据结构与操作命令
    • String：GET/SET、INCR/DECR。
    • Hash：HGET/HSET、HGETALL。
    • List：LPUSH/RPUSH、LPOP/RPOP。
    • Set：SADD/SMEMBERS、集合运算。
    • Sorted Set：ZADD/ZRANGE、按分数排序。
  • 2.3 持久化机制
    • RDB：时间点快照。
    • AOF：记录所有写操作。
  • 2.4 单线程架构
    • 核心操作由单线程执行，避免线程切换开销。
  • 2.5 哨兵和集群机制
    • 哨兵：高可用方案。
    • 集群：数据分片与自动管理。

3. 分布式数据库CAP原理

  • 3.1 CAP简介
    • C（一致性）、A（可用性）、P（分区容忍性）。
  • 3.2 CAP理论
    • 分区常态下需选择A或C。
    • CA：单点集群，强一致性。
    • CP：一致性优先，性能较低。
    • AP：可用性优先，最终一致性。

4. Redis的安装与配置

  • 4.1 安装
    • 下载：官网或镜像（如华为云）。
    • 安装步骤：解压、编译。
  • 4.2 启动
    • 默认启动、指定配置文件、开机自启。
  • 4.3 客户端
    • 命令行：redis-cli。
    • 图形化：Redis Desktop Manager。
  • 4.4 配置
    • 参数：maxmemory、logfile、port、bind、appendonly、database。

5. Redis基本操作

  • 5.1 查询命令
    • 官网查询、HELP命令。
  • 5.2 Key键
    • 特性：命名规范、二进制安全、过期时间。
    • 命令：DEL、EXPIRE、TTL、SCAN。
  • 5.3 通用命令
    • DEL、DUMP、EXISTS、EXPIRE、TTL、SCAN、TYPE。
  • 5.4 数据库指令
    • FLUSHDB、FLUSHALL、DBSIZE、MOVE、SELECT。
  • 5.5 客户端命令
    • CLIENT LIST、CLIENT SETNAME、CLIENT PAUSE。

6. Redis数据类型

  • 6.1 基础类型
    • String：最大512MB，命令：SET、GET、INCR。
    • Hash：字段-值映射，命令：HSET、HGET。
    • List：有序、允许重复，命令：LPUSH、LRANGE。
    • Set：无序唯一，命令：SADD、SMEMBERS。
    • Sorted Set：带分数排序，命令：ZADD、ZRANGE。
  • 6.2 特殊类型
    • Bitmap：位操作，命令：SETBIT、BITCOUNT。
    • HyperLogLog：基数统计，命令：PFADD、PFCOUNT。
    • Geo：地理位置，命令：GEOADD、GEODIST。

7. Redis发布订阅

  • 7.1 基本概念
    • 发布者、订阅者、频道。
    • 应用场景：聊天系统、配置更新、订单通知、消息队列。
  • 7.2 命令
    • PSUBSCRIBE、PUBSUB、PUBLISH、SUBSCRIBE。
  • 7.3 案例
    • 订阅者接收消息、发布者发送消息。

8. Stream消息队列

  • 8.1 原理
    • 消息链、消费组、游标、Pending_ids。
  • 8.2 命令
    • XADD、XREAD、XGROUP CREATE、XACK。
  • 8.3 使用
    • 创建消息、消费消息、创建消费组。

9. Redis事务

  • 9.1 特性
    • 多命令原子性，不保证事务整体原子性。
    • 阶段：开启（MULTI）、命令入队、执行（EXEC）。
    • 应用场景：批量操作、数据库迁移、分布式锁。
  • 9.2 命令
    • MULTI、EXEC、DISCARD、WATCH。
  • 9.3 案例
    • 正常事务、取消事务、监控键。

10. Redis布隆过滤器

  • 10.1 原理
    • 位数组+哈希函数，误判率存在。
  • 10.2 应用场景
    • 垃圾邮件过滤、缓存穿透防护、大数据去重、数据库查询优化。
  • 10.3 安装与使用
    • 直接编译、Docker安装，命令：bf.add、bf.exists。

11. Redis Lua脚本

  • 11.1 优势
    • 减少网络开销、原子性、复用性。
  • 11.2 场景
    • 事务处理、复杂数据操作、性能优化。
  • 11.3 命令
    • EVAL、SCRIPT LOAD、EVALSHA、SCRIPT FLUSH。

12. Redis管道技术（Pipeline）

  • 12.1 优势
    • 减少RTT，批量操作。
  • 12.2 场景
    • 事务处理、批量操作、缓存预热、高并发。
  • 12.3 对比
    • Pipeline与Script的性能差异。

13. Jedis客户端

  • 13.1 功能
    • 支持单机、分片、集群、哨兵模式。
  • 13.2 API
    • String、Key、Hash、List、Set、Sorted Set操作。
  • 13.3 高级操作
    • 批处理（MSET）、事务、JedisPool连接池。
  • 13.4 JedisPool
    • 初始化、获取/归还连接、关闭池。

二、Redis应用

1. 引言

  • 缓存的重要性：提升性能、减少数据库压力、用户体验。
  • Spring Boot缓存支持：透明化、注解简化管理。
  • Redis优势：存取速度、数据类型、原子性、功能扩展。

2. Spring Boot默认缓存管理

  • 2.1 基础环境搭建
    • 创建项目、依赖引入、实体类、Repository、业务层、Web层、配置。
  • 2.2 缓存体验
    • @EnableCaching注解、@Cacheable注解、测试。

3. Spring Boot缓存注解

  • 3.1 @EnableCaching
    • 开启缓存支持。
  • 3.2 @Cacheable
    • 属性：value、key、condition、unless。
  • 3.3 @CachePut
    • 更新缓存，与@Cacheable属性一致。
  • 3.4 @CacheEvict
    • 清除缓存，属性：allEntries、beforeInvocation。
  • 3.5 @Caching
    • 复合注解，组合多个缓存操作。
  • 3.6 @CacheConfig
    • 类级公共配置。

4. Spring Boot整合Redis

  • 4.1 缓存组件
    • 默认使用Simple，需配置Redis依赖。
  • 4.2 整合步骤
    • 添加依赖、配置连接参数、实体类映射、Repository操作、测试。

5. Redis缓存实现

  • 5.1 注解实现
    • @Cacheable、@CacheEvict、序列化（Serializable）。
  • 5.2 API实现
    • RedisTemplate操作、序列化配置。
  • 5.3 连接池配置
    • Lettuce参数：max-active、max-idle等。

6. 自定义序列化机制

  • 6.1 RedisTemplate
    • 自定义JSON序列化（Jackson2JsonRedisSerializer）。
  • 6.2 RedisCacheManager
    • 自定义配置，使用JSON序列化。

7. StringRedisTemplate操作

  • 7.1 String操作
    • SET、GET、SETEX、SETNX。
  • 7.2 Key操作
    • EXPIRE、KEYS。
  • 7.3 Hash操作
    • HSET、HGET、HGETALL。
  • 7.4 List操作
    • LPUSH、RPUSH、LRANGE。
  • 7.5 Set操作
    • SADD、SMEMBERS、SRANDMEMBER。
  • 7.6 Sorted Set操作
    • ZADD、ZRANGE、ZSCORE。

8. 本章小结

  • 缓存注解的使用、自定义序列化、Redis数据类型操作。

三、Redis分布式缓存

1. 引言

  • 目标：解决数据丢失、并发能力、故障恢复、存储能力问题。

2. Redis RDB持久化

  • 2.1 触发策略
    • 自动（save指令）、手动（SAVE/BGSAVE）。
  • 2.2 配置
    • dir、dbfilename、rdbcompression、rdbchecksum。
  • 2.3 工作原理
    • COW机制，父进程fork子进程生成RDB文件。
  • 2.4 优劣势
    • 优势：速度快、恢复快。
    • 劣势：实时性差、数据丢失风险。

3. Redis AOF持久化

  • 3.1 策略
    • always、everysec、no。
  • 3.2 配置
    • appendonly、appendfilename、appendfsync。
  • 3.3 重写
    • 原理：生成新AOF文件。
    • 触发：手动（BGREWRITEAOF）或自动。
  • 3.4 优劣势
    • 优势：安全性高。
    • 劣势：文件大、恢复慢。

4. Redis主从复制

  • 4.1 架构
    • 一主多从，读写分离。
  • 4.2 配置
    • REPLICAOF命令、配置文件。
  • 4.3 问题
    • 主从宕机后的同步、故障转移。

5. Redis哨兵模式

  • 5.1 原理
    • 监控、客观下线、故障转移。
  • 5.2 配置
    • sentinel monitor、sentinel down-after-milliseconds、quorum。
  • 5.3 优劣势
    • 优势：自动故障转移。
    • 劣势：写延迟、哨兵单点风险。

6. Redis分片集群

  • 6.1 架构
    • Master-Slave结构，数据分片（16384槽）。
  • 6.2 操作
    • 数据分布、故障转移、扩容缩容。
  • 6.3 配置
    • 创建集群、哈希槽分配、客户端路由支持。

7. RedisTemplate访问集群

  • 7.1 配置
    • Sentinel或分片集群参数、读写分离策略。
  • 7.2 操作
    • StringRedisTemplate读写数据。

8. 总结

  • 持久化、主从+哨兵、分片集群的综合方案。

7. 模块与包

七、Redis原理

1. 数据结构

  • SDS：字符串结构体。
  • IntSet：整数集合。
  • Dict：哈希表（扩容、rehash）。
  • ZipList/QuickList：压缩列表、快速链表。
  • SkipList：跳跃表。
  • RedisObject：对象类型与编码。

2. 数据类型原理

  • String：RAW/EMBSTR/INT编码。
  • List：QuickList结构。
  • Hash：ZipList/Dict。
  • Set：IntSet/Dict。
  • Sorted Set：ZipList/SkipList。

3. 网络模型

  • IO模型：阻塞、非阻塞、epoll。
  • Redis实现：事件循环、epoll驱动。

4. 通信协议（RESP）

  • 格式：简单字符串、错误、整数、批量字符串、数组。

5. 内存策略

  • 过期策略：惰性删除、定期删除。
  • 淘汰策略：LRU、LFU、随机、noeviction。

八、Redis大厂面试

1. 应用场景

  • 缓存、会话、消息队列、分布式锁、社交、实时分析、排行榜、限流。

2. 数据类型

  • String、List、Set、Hash、Sorted Set、Bitmap、HyperLogLog、Geo、Stream。

3. 过期键删除策略

  • 惰性删除、定期删除、组合策略。

4. 内存淘汰策略

  • volatile-lru、allkeys-lfu等。

5. 分布式锁实现

  • SETNX+Lua脚本、误删防护、自动续期。

6. 线程模型

  • 单线程（6.0前）、多线程IO（6.0后）。

7. 集群模式

  • 主从复制、哨兵、Redis Cluster（哈希槽、路由）。

8. 集群槽数16384的原因

  • 心跳包体积、节点数量限制、压缩效率。

9. 缓存问题

  • 雪崩（差异化过期）、击穿（锁）、穿透（布隆过滤器）。
