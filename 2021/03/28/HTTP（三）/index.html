HTTP（三）

1. HTTP特点

1.1 优点

  • 灵活可扩展

    对报文没有过多的限制，传输数据也没有限制，文本、图片、视频等都可以。

    传输的实体数据可压缩、缓存；

    可分段获取数据；

    支持身份认证；

    支持国际化语言。

  • 相对可靠

    HTTP是基于TCP/IP传输的，TCP/IP是可靠的连接传输协议（相对而言可靠）。

  • 请求-应答

    HTTP使用请求-应答的通信模式。

  • 无状态

    是无状态的协议（不会有记录，每次都是建立连接、发送/接收数据）。

  • 应用层协议

    没有太多局限性，如果不在意性能，可以传输任何文件。

    例如：FTP只传输文件，SMTP只发送邮件等

1.2 缺点

  1. 通信使用明文（不加密），容易被窃取

    • 这个是TCP/IP协议族的工作机制，本身就可能被窃听

    • 加密的方法

      • 使用 SSL或TLS 进行加密。使用SSL建立安全通信线路后，就可以在这条线路上进行HTTP通信，也被称为HTTPS。

      • 内容加密。将HTTP报文进行加密，需要客户端和服务端有相同的解密机制，但是这种方式仍有被篡改的风险。

  2. 不验证通信双方的身份，有可能伪装

    • 任何人都可以发起请求。HTTP协议通信时，不存在确认通信方的机制，如果没有限制IP、端口号等，对于任何请求都会做出响应。

    • 使用证书校验。由可信赖的第三方颁发证书，证明通信的服务器是可信赖的。

  3. 无法验证报文的完整性，有可能被篡改

    • 接收到的内容可能有误。没有办法确认发出的请求/相应和接受到的请求/相应是相同的。

    • 通常使用MD5和SHA-1等方法校验，用来确认文件的数字签名。

2. HTTPS

HTTP + 加密 + 认证 + 完整性保护 = HTTPS

HTTPS并不是新协议，HTTPS就是身披SSL协议外壳的HTTP。

HTTPS使用混合加密的方式，先用公钥把共享秘钥进行加密，然后传输；确保交换的秘钥是安全的情况下，使用共享秘钥进行通信。

2.2 缺点

  1. 通信慢

    除了HTTP请求外，还要进行SSL通信，所以通信量会不可避免的增加。

  2. 处理速度慢

    SSL需要加密处理，在客户端和服务端都需要加密、解密的操作，消耗硬件资源。
