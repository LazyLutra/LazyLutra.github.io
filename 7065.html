常用命令

clone指定分支

git clone -b 分支名 git地址

用户操作

  1. 查看当前用户名和邮箱：

  2. 切换用户名和邮箱：

  3. 查看操作记录： git reflog

上传流程

  1. 查看文件修改状态： git status
  2. 文件夹下所有修改提交到暂存区： git add .
  3. 文件夹下提交单个文件到暂存区： git add 文件名称
  4. 提交并填写提交信息： git commit -m "这里填写提交信息"
  5. 推送到Git仓库上的默认分支： git push

分支操作

  1. 查看所有分支： git branch -a
  2. 查看当前所属分支： git branch
  3. 切换分支： git checkout 分支名称

版本恢复

使用git reset

原理： 修改HEAD指向的位置，抛弃该位置之后的全部修改

  1. 使用 git log 查看版本号，或者在 git 页面上查看版本号

  2. 使用 git reset --hard 目标版本号

    例如：git reset --hard ee8809ce

  3. 再次使用 git log 查看当前版本是否为目标版本

  4. 使用 git push -f 强制提交（因为本地 HEAD 指向的版本比远程库旧，所以需要强制提交）

其他操作

  1. 将暂存区(index)内的文件全部删除：只删除索引，不删除原文件和修改的部分。（HEAD为当前分支 ）
    git reset HEAD
  2. 保存凭证，在git根目录下使用：git config --global credential.helper store

当前分支落后于主分钟版本

git合并分支时当前分支落后于主分支

常见问题

IDEA clone 报错

缓存溢出

IDEA 使用 git clone 项目报错：

  1. 原因一：缓存区溢出，重新设置大小

    git config http.postBuffer 524288000

  2. 原因二：下载速度慢

    git config --global http.lowSpeedLimit 0

    git config --global http.lowSpeedTime 999999

  3. 其他解决方法

    先浅克隆到本地，然后远程库拉取

pull失败如何解决

提示远程仓库不存在

  1. 移除本地存储的仓库地址

    git remote rm origin

  2. 获取 ssh 地址，重新与远程仓库建立关联

    git remote add origin 仓库地址

  3. 建立成功后，重新查看 branch 情况

    git fetch origin

  4. 检查远程仓库地址

    这一步会出现两个相同的 git 地址，末尾分别是 (fetch) 和 (push)

    git remote -v

  5. 查看当前分支

    git branch

hexo d 报错

OpenSSL SSL_read: Connection was reset, errno 10054

可能是网络不稳定造成的，如果排除网络问题，可以解除 SSL 验证后再次尝试：

git config --global http.sslVerify "false"

OpenSSL SSL_connect: SSL_ERROR_SYSCALL in connection to github.com:443

有可能是网络或者 HTTP 代理的问题，尝试以下命令：

git config --global --unset http.proxy

Changes not staged for commit

当修改文件名称大小写时可能会出现这个错误，例如本地文件名称为：TestFile.java，而库中的文件名称为：TESTFile.java。

  1. 此时需要先把出错的文件内容拷贝出来，删除错误的文件并 push。

  2. 然后新建文件，粘贴好内容后 push 即可解决。

IDEA 中执行第二步时可能不会提示该文件需要 add，此时删除并再次新建文件就会弹出 add 的提示。
