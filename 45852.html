SpringBoot（十）：配置文件详解

发表于2022-04-30|更新于2022-05-17|SpringBoot

配置文件区别

  1. 提供了两种类型：properties 和 yml/yaml

  2. 默认配置文件名称：application

  3. 同一级目录下优先级位：properteis > yml > yaml

  4. 如果低优先级文件有高优先级文件没有的配置信息，也会生效

YML/YAML 语法和格式

语法要求

  1. 大小写敏感

  2. 数据值前必须有空格作为分隔符

  3. 使用空格表示层级关系，相同缩进为一级

  4. 不允许使用 tab 缩进，必须使用空格（idea使用tab时默认产生4个空格）

  5. 缩进空格数不重要，只要相同层级的元素左侧对齐

  6. 使用 # 作为行注释

  7. 使用 ${key} 获取数据

数据格式

  • Map

  • 数组（使用 - 表示每个元素）

  • 常量

获取配置文件内容

application.yml 配置的数据

@Value

需要手动单独配置每个属性注入，属性太多时比较繁琐。

Environment

只注入一次即可，使用时需要指定属性名。

@ConfigurationProperties

配置内容和对象绑定。

注解 @ConfigurationProperties 必须要指定前缀，否则会识别到其他同名的属性。

profile

动态切换开发、测试、生产环境的配置文件。

yml 一个文件下配置多个环境时，必须使用 --- 分隔。

配置和激活方式

  • 配置文件

    此处指定生效的配置文件为 dev 后缀。

    文件名为：application-dev.yml

  • 虚拟机参数

    文件名为：application.yml

    虚拟机参数：-Dspring.profiles.active=test1

  • 命令行参数

    文件名为：application.yml

    命令行参数或者在命令行启动项目 jar 包时使用该命令：--spring.profiles.active=test2。

内部配置加载顺序

注意：
1. 同级目录下存在两个同名配置文件时，优先读取 properties。
2. 存在多个配置文件时为互补的关系，高优先级文件中没有的部分会由优先级文件的配置补充。

按照优先级由高到低排列：

  1. file:./config/ 当前项目下的 config 目录
  2. file:./ 当前项目的根目录
  3. classpath:./config/ classpath 下的 config 目录
  4. classpath: classpath 的根目录
