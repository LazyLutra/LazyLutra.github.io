数据类型

数据类型总结
类型 存储需求 取值范围
整形 int 4字节 -2,147,487,648 ~ 2,147,487,647（正好超过20亿）
short 2字节 -32,768 ~ 32,767
long 8字节 -9,223,372,036,854,775,808 ~ 9,223,372,036,854,775,807
byte 1字节 -128 ~ 127
浮点型 float 4字节 有效位数 6 ~ 7 位
double 8字节 有效位数 15 位
char char 与编码格式有关，unicode为 2 字节 \u0000（十进制等效值为 0） ~ 最大值是 \uffff（即为 65535）
boolean boolean 没有规定大小，单个 boolean 占 1 字节，数组 boolean[] 中每个元素占 4 字节 只有 true 和 false

对象与类

对象

所有的 Java 对象都是在堆中构造的。

  • 对象的行为（behavior)
    可以对对象施加哪些操作，或可以对对象施加哪些方法？
  • 对象的状态（state )
    当施加那些方法时，对象如何响应？
  • 对象标识（identity )
    如何辨别具有相同行为与状态的不同对象

方法不要返回可变对象

可变对象：对象被创建后，状态可以被改变
不可变对象：对象被创建后，状态不可变

对象构造

重载

多个方法有相同的名字，不同的参数。

默认值

构造器中如果没有显式赋值，则会使用默认值：

  • boolean：false
  • 数值：0
  • 对象引用：null
无参构造器

创建类时默认提供一个无参构造器，所有实例域的值会被设置为默认值；

如果类中创建了一个构造器，但是没有无参构造器，则会在 new 时报错。

类

类之间的关系

  • 依赖（uses-a）
  • 聚合（has-a）
    例如，一个 Order 对象包含了 Item 对象
  • 继承（is-a）

构造器

  • 构造器与类同名
  • 每个类可以有一个或以上的构造器
  • 构造器可以有 0 个、1 个或者多个参数
  • 没有返回值
  • 构造器总是伴随 new 操作一起使用

方法参数

Java 总是使用按值调用，方法得到的是参数值的拷贝，并且不能修改传递给方法的参数变量内容。
对象引用是按值传递的。

方法参数具有两种类型：

  1. 基本数据类型
  2. 对象类型

方法参数使用情况：

  1. 一个方法不能修改基本数据类型的参数（数值型或者布尔型）
  2. 一个方法可以改变对象参数的状态
  3. 一个方法不能让对象参数引用一个新的对象

继承

super 不是对象的引用，只是告诉编译器需要调用父类方法或者父类构造器。

this 作用是引用隐式参数和调用该类的其他构造器。

Object 类

是所有类的父类，如果没有明确继承某个类，就会自动继承 Object。

equals()

  1. 自反性：对于任何非空引用 x，x.equals(x) 应该返回 true。
  2. 对称性: 对于任何引用 x 和 y，当且仅当 y.equals(x) 返回 true，x.equals(y) 也应该返回 true。
  3. 传递性：对于任何引用 x、 y 和 z，如果 x.equals(y) 返回 true，y.equals(z) 返回 true，x.equals(z) 也应该返回 true。
  4. 一致性：如果 x 和 y 引用的对象没有发生变化，反复调用 x.equals(y) 应该返回同样的结果。
  5. 对于任意非空引用 x，x.equals(null) 应该返回 false。

hashCode()

每个对象都有一个默认 hash 值，如果是两个不同的对象，则 x.hashCode() 和 y.hashCode() 基本不会相同。

内部类

把类定义在另一个类的内部，就是内部类。

访问规则：

  • 可以访问外部类的所有成员，包括私有的。

  • 外部类必须创建对象才能访问内部类的成员。

作用

概括：封装、解耦、提高内聚

  1. 满足封装性

  2. 间接实现多继承（就是一个类能有多个内部类，每个内部类单继承一个不同的类即可勉强算多继承）

  3. 匿名内部类实现回调（其实就是把匿名内部类的实现对象作为实参直接传给收接口型形参的调用者，从而实现回调）

  4. 解决继承 + 实现时碰到的方法同名问题

  5. 通过静态内部类实现线程安全的单例

分类

  1. 成员内部类
    成员内部类——就是位于外部类成员位置的类。
    特点：可以使用外部类中所有的成员变量和成员方法（包括private的）。

  2. 局部内部类
    局部内部类——就是定义在一个方法或者一个作用域里面的类。
    特点：主要是作用域发生了变化，只能在自身所在方法和属性中被使用。

  3. 静态内部类
    我们所知道static是不能用来修饰类的,但是成员内部类可以看做外部类中的一个成员,所以可以用static修饰,这种用static修饰的内部类我们称作静态内部类,也称作嵌套内部类。
    特点：不能使用外部类的非static成员变量和成员方法。

  4. 匿名内部类
    一个没有名字的类，是内部类的简化写法。

局部内部类

在方法或者一个作用域中定义的类，如果需要访问局部变量，该变量必须使用 final 修饰。

局部变量是随着方法的调用而调用，使用完毕就消失，而堆内存的数据并不会立即消失。

所以，堆内存还是用该变量，而该变量已经没有了。为了让该值还存在，就加final修饰。

当我们使用final修饰变量后，堆内存直接存储的是值，而不是变量名。
